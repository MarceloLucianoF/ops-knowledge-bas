# Versionamento (SemVer)

## SemVer
- **MAJOR.MINOR.PATCH**
- MAJOR: breaking changes
- MINOR: feature compatível
- PATCH: bugfix compatível

## Tags
- Use tags Git para releases:
  - `v1.4.2`

## Boas práticas
- Changelog por release
- Tag imutável e auditável
- Artefatos (imagem Docker) versionados com a mesma tag
